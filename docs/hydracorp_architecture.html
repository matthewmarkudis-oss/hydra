Core Design Principle

LLM agents never trade directly. They adjust the HydraConfig parameters that control how RL agents are trained. The RL layer (PPO, SAC, RecurrentPPO, 9 rule-based agents) handles all actual trading decisions during backtesting. The corporation layer provides strategic oversight, regression prevention, and external intelligence.

System Layers

HydraCorp separates concerns into four layers. Information flows downward as config changes and upward as performance metrics.

CEO Layer (You)

The CEO Dashboard at localhost:5050 shows portfolio value in CAD, agent performance, risk gauges, alerts, and benchmark comparison. All dollar amounts are scaled from the RL engine's internal $100K to your real $2,500 starting capital. You approve or reject config proposals, promote shadow configs, and set strategic direction.

Corporation Layer (8 Agents)

The Chief of Staff orchestrates a 6-phase cycle: pre-flight validation, pipeline execution, strategy analysis, intelligence gathering, shadow comparison, and CEO briefing. Four agents use LLMs (with rule-based fallbacks); four are pure Python. All communicate through typed messages and shared state persisted to JSON.

Config Interface (HydraConfig YAML)

The only communication channel between corp and RL layers. Corp agents propose patches to RewardConfig, EnvConfig, and PoolConfig sections. Patches are validated by the Senior Dev before application. Config hashes (SHA-256) track what's been tried and prevent regressions.

RL Layer (Hydra Engine)

Population-based training across 30 generations. Three learning agents (PPO, SAC, RecurrentPPO) evolve alongside static snapshots and rule-based agents. Five protocols govern the pipeline: CHIMERA (diagnostics), PROMETHEUS (weight rebalancing), ELEOS (conviction scoring), ATHENA (walk-forward validation), and KRONOS (curriculum scheduling).

Execution Cycle

Each corporation cycle runs these six phases in sequence. The graph executor supports conditional routing — phases can be skipped based on pre-flight results or pipeline errors.

1

Pre-Flight

Senior Dev validates the proposed config against the blacklist (SHA-256 hash lookup), checks for recent duplicates, and reviews any pending patches for safety constraint violations. Hardware Optimizer detects CPU/GPU config and recommends optimal n_envs and cpu_workers. If the config is blacklisted, the cycle terminates early.

Senior Dev Hardware Optimizer
2

Pipeline Execution

Runs the full Hydra training pipeline: data preparation (real Alpaca bars or synthetic), environment building, population-based training (30 generations, 100 episodes each), evaluation (VectorBT cross-validation), and ATHENA walk-forward validation. Results are recorded in corporation state.

PipelineOrchestrator
3

Post-Analysis

Hedge Fund Director analyzes results and proposes config patches (e.g., reduce transaction penalty, tighten drawdown limits). Contrarian fires if any agent passed validation or returns are suspiciously high — challenges results for hidden fragility and proposes stress-test configs.

Hedge Fund Director Contrarian
4

Intelligence

Geopolitics Expert fetches headlines from NewsAPI, Finnhub, or Yahoo RSS and classifies the market regime (risk_on / risk_off / crisis). Innovation Scout searches for new tools and libraries relevant to the RL trading stack. Both run on schedules (daily and weekly) unless forced.

Geopolitics Expert Innovation Scout
5

Shadow Comparison

If a shadow config has been set (from Hedge Fund Director or Contrarian proposals), the Shadow Trader runs a parallel pipeline with the experimental config and compares results against the primary. After 3 consecutive shadow wins, promotion is recommended.

Shadow Trader
6

CEO Briefing

Aggregates all phase outputs into a CEO-friendly summary: portfolio value (CAD), total return, best agent, benchmark comparison, market regime, pending proposals, and recent alerts. Displayed on the dashboard.

Chief of Staff

Central Coordinator & Router
Pure Python $0 / run

The central nervous system of the corporation. Orchestrates the full 6-phase cycle, routes events to the correct division, enforces run schedules, and aggregates reports for the CEO dashboard. Wraps the PipelineOrchestrator with pre/post hooks.

Key Responsibilities

  • Execute 4-phase corporation cycle

  • Generate CEO briefing with portfolio metrics
  • Check shadow promotion logic
  • Record pipeline results to state

Decisions Logged

  • config_veto — blocked by blacklist
  • pipeline_complete — successful run
  • pipeline_failed — error during run
  • shadow_promotion_recommended
🔐

Senior Dev

Config Regression Prevention
Pure Python $0 / run

The guardrail. Maintains a blacklist of failed config hashes (SHA-256) and validates every proposed config before pipeline runs. Detects regressions by comparing fitness against historical bests. Reviews patches from the Hedge Fund Director for safety constraint violations.

Safety Constraints Enforced

  • transaction_penalty ≥ 0.005
  • drawdown_penalty ≥ 0.1
  • reward_scale between 10 and 500
  • max_position ≤ 60%
  • max_drawdown ≤ 40%

Regression Detection

  • Config hash lookup against blacklist
  • Duplicate detection (last 10 runs)
  • If return < 50% of historical best: blacklist
  • Bulk init from meta_optimize.jsonl
⚙

Hardware Optimizer

CPU/GPU Configuration Tuning
Pure Python $0 / run

Detects CPU, GPU (DirectML), and RAM configuration. Recommends optimal n_envs and cpu_workers based on available hardware. Detects AMD Smart Access Memory (SAM/ReBAR) via Windows registry. Tracks per-generation training times for trend analysis.

Auto-Tuning Rules

  • 12+ cores: 8 CPU workers
  • 8+ cores: 6 CPU workers
  • 16GB+ GPU: 8 parallel envs
  • 8GB+ GPU: 4 parallel envs

Detection Capabilities

  • AMD/Intel CPU identification
  • DirectML GPU memory & type
  • SAM/ReBAR via registry
  • Total RAM via WMI

Strategy Division

Post-pipeline analysis and config optimization proposals. LLM-powered with rule-based fallbacks.

📈

Hedge Fund Director

Strategic Config Advisor
LLM (Claude Haiku) Every Pipeline Run

Analyzes pipeline results (CHIMERA fitness, Sharpe ratios, drawdowns, win rates) and proposes specific parameter changes to RewardConfig and PoolConfig. Generates strategy memos with confidence scores. Proposals are submitted through the message bus to Senior Dev for safety review before application.

Rule-Based Fallback Logic

  • No agents passed: reduce transaction_penalty, increase reward_scale
  • Negative returns: tighten max_drawdown, increase drawdown_penalty
  • Underperforming benchmark: reduce holding_penalty
  • Good results (>5%): no major changes

Output

  • Strategy memo (2-3 sentences)
  • JSON config patch (validated by Pydantic)
  • Confidence score (0.0 – 1.0)
  • Risk assessment (low/medium/high)
⚠

Contrarian

Taleb-Style Adversarial Reviewer
LLM (Claude Haiku) Conditional Trigger

The mandatory skeptic. Inspired by Nassim Taleb's philosophy, this agent fires only when results look good — and its job is to find what's wrong. Checks for overfitting, inflated Sharpe ratios, unrealistic cost assumptions, and period-specific luck. Proposes stress-test configs with tighter constraints.

Trigger Conditions

  • Any agent passed ATHENA validation
  • Best return exceeds threshold (>0.5)
  • Pipeline declares convergence
  • Does NOT fire on bad results

Fragility Checks

  • Only 1 of many agents passed → fragile outlier
  • Return >20% → likely overfitting
  • Transaction penalty <0.01 → unrealistic
  • Max position ≥50% → concentration risk

Intelligence Division

External data ingestion and technology scouting. LLM-powered with schedule-based execution.

🌎

Geopolitics Expert

News-Driven Regime Classification
LLM (Claude Haiku) Daily Free APIs

Fetches financial headlines from NewsAPI, Finnhub, or Yahoo Finance RSS (cascading fallback). Classifies the macro regime as risk_on, risk_off, or crisis. Output feeds into config adjustments — not trade signals. For example, a crisis regime might tighten drawdown limits and reduce position sizes in the next training cycle.

News Sources

  • NewsAPI — 100 req/day (free tier)
  • Finnhub — 60 req/min (free tier)
  • Yahoo RSS — unlimited (no key)
  • Falls back silently through the chain

Keyword Analysis (Fallback)

  • Crisis: crash, panic, recession, contagion
  • Risk-off: fear, sell-off, hawkish, tariff, war
  • Risk-on: rally, surge, bull, rate cut, recovery
  • Counts keywords, highest bucket wins
🔍

Innovation Scout

Tool & Technique Discovery
LLM (Claude Haiku) Weekly

Searches for new tools, libraries, and techniques relevant to RL trading systems. Produces structured "innovation briefs" with relevance scores, integration effort estimates, and priority levels. Does NOT auto-install anything — proposals are filed for CEO review on the dashboard.

Focus Areas

  • RL trading frameworks (FinRL, etc.)
  • Stable-Baselines3 extensions
  • VectorBT and backtesting tools
  • Reward engineering techniques

Static Fallback Recommendations

  • FinRL — RL trading framework
  • SB3-Contrib — RecurrentPPO, TRPO
  • Optuna — hyperparameter optimization

Support

A/B config testing via dual pipeline execution.

👀

Shadow Trader

Dual Pipeline A/B Testing
Pure Python $0 / run When Shadow Config Set

Runs the primary (current best) config alongside an experimental shadow config proposed by the Hedge Fund Director or Contrarian. Both pipelines share cached market data. Tracks consecutive wins and recommends promotion after 3 straight shadow victories. Used for backtesting and training comparison only.

Comparison Logic

  • Runs both configs on same data
  • Compares best_return from each
  • Tracks win/loss history (last 20)
  • Calculates shadow win rate

Promotion Rules

  • 3 consecutive shadow wins required
  • Sends promotion proposal (priority 4)
  • CEO must approve on dashboard
  • Shadow cleared after promotion/rejection

Data Pipeline

The system supports both synthetic data (for development and testing) and real historical data from Alpaca's free IEX feed (for meaningful backtesting). 

Synthetic Mode (default):
  Random GBM prices → indicators (16 features) → StateBuilder (15N+5 obs)

Real Data Mode (--real-data):
  Alpaca IEX 5-min bars → Parquet cache → indicators (16 features) → StateBuilder (15N+5 obs)
  |
  +-- 10 tickers: NVDA, TSLA, AMD, MARA, COIN, META, AMZN, GOOGL, NFLX, SQ
  +-- Lookback:   252 trading days (~1 year)
  +-- Interval:   5-minute bars
  +-- Bars/ticker: ~19,000-22,000
  +-- Cache:      data/bars_cache/{ticker}_{timeframe}_{start}_{end}.parquet
  +-- Cost:       $0 (Alpaca IEX free tier)

Each episode = 390 bars = 1 trading week of 5-min data
30 generations x 100 episodes = 3,000 simulated weeks on real price history

Design Principles

Graceful Degradation

  • Every LLM agent has a rule-based fallback
  • News APIs cascade: NewsAPI → Finnhub → RSS
  • No API key? System runs fully on heuristics
  • SQ has no IEX data? Falls back to synthetic

Safety First

  • Senior Dev validates every config before execution
  • Contrarian challenges every positive result
  • Blacklist prevents regression to failed configs
  • All proposals require review before application

Full Auditability

  • Every decision logged to JSONL with timestamps
  • Message bus tracks all inter-agent communication
  • Config hashes enable exact experiment reproduction
  • Proposals tracked from submission to resolution

Cost Discipline

  • Pure Python for all frequent operations
  • LLMs only for infrequent strategic analysis
  • Monthly budget cap in config ($10 default)
  • Free APIs for all external data sources
