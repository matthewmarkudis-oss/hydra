RL Agents (Hydra Core)

▼

These agents learn continuous portfolio allocation actions [-1, +1] per stock via Stable-Baselines3. All agents share one portfolio, observe the same state, and their actions are aggregated via weighted mean.

📖 Learning Agents
🧠
PPO Agent (Default)
ppo_1 — Proximal Policy Optimization
On-policy actor-critic with clipped surrogate objective. Primary learning agent — stable gradients, good sample efficiency for continuous control.
net: [256, 256] lr: 3e-4 clip: 0.2 ent: 0.01 GAE: 0.95 n_steps: 2048
🔬
PPO Agent (Exploratory)
ppo_2 — Higher entropy, faster learning
Variant with smaller network, higher learning rate, and 2x entropy bonus. Explores more aggressively — finds unconventional strategies that the default PPO misses.
net: [128, 128] lr: 1e-3 ent: 0.02 clip: 0.2 GAE: 0.95
🎯
TD3 Agent
td3_1 — Twin Delayed DDPG
Off-policy deterministic actor with twin critics and delayed policy updates. Better for precise continuous actions. Uses replay buffer (100K transitions).
net: [256, 256] lr: 3e-4 buffer: 100K tau: 0.005 policy_delay: 2
🧬
Recurrent PPO Agent
rppo_1 — PPO + LSTM Memory
PPO with LSTM layers for temporal pattern recognition. Maintains hidden state across steps — can learn sequential dependencies that MLP agents miss (e.g., multi-day trends).
LSTM: 128 hidden net: [256] lr: 3e-4 1 LSTM layer n_steps: 2048
⚡
A2C Agent
a2c — Advantage Actor-Critic
Synchronous A2C with GAE. Simpler than PPO (no clipping), faster per step. Available in pool but typically outperformed by PPO variants.
lr: 7e-4 n_steps: 5 vf: 0.25 ent: 0.01
🔥
SAC Agent
sac — Soft Actor-Critic
Maximum entropy off-policy with automatic entropy tuning. Excellent exploration but historically benched by CHIMERA in this system (replaced by TD3 on warm-start).
net: [256, 256] lr: 3e-4 buffer: 100K ent: auto benched
❄ Special Agent Types
📸
Static Agent (Frozen Snapshot)
Immutable copy of trained agent
Created by promote_top(k) — freezes a learning agent's weights as a static baseline. Cannot learn. Used for performance comparison across generations. File-based + state_dict copy with CPU-safe loading.
frozen=True CPU-safe load weight checksum
📜
Rule-Based Agent (Wrapper)
Adapter for trading_agents strategies
Wraps existing BaseAgent strategies (ALPHA Momentum, BETA Mean Reversion) and translates their TradeSignal output into continuous [-1, +1] actions. Always frozen (no gradient learning). Provides non-ML diversity in the pool.
alpha_rule beta_rule frozen=True

🏗 Agent Pool — Population Management

AgentPool manages the diverse population. Actions from all agents are aggregated via weighted mean into a single portfolio decision.

1Collect actions from all agents for same observationcollect_actions()
2Weighted average → single action vector, clipped to [-1, 1]aggregate_actions()
3Promote top-K learning agents → freeze as static snapshotspromote_top(k=2)
4Demote bottom-K static agents → remove from pooldemote_bottom(k=1)
5Update rankings from eval scores (higher = better)update_rankings()

🔄 Generation Loop (Inside Phase 3)

Each generation runs the following steps for all agents in the pool. Managed by PopulationTrainer.

1Run multi-agent episodes (shared obs, aggregated actions, shared reward)Trainer.train_episodes()
2SB3 gradient updates per learning agent (.learn())_train_agents_on_env()
3Deterministic evaluation episodes on val_env_evaluate_agents()
4CHIMERA diagnostics: detect overfitting, drawdown, stagnation → mutation recommendationsDiagnosticEngine.diagnose()
5Auto reward tuning: map mutations → bounded ±20% reward param adjustments (every 5 gens)AutoRewardTuner.apply_mutations()
6ELEOS conviction calibration: Bayesian trust gating for new/unproven agentsConvictionCalibrator.update()
7PROMETHEUS competition: composite score → EMA-smoothed weight rebalancingCompetitionRebalancer.rebalance()
8Promote top-K learning agents → frozen static snapshotspool.promote_top()
9Demote bottom-K static agents → remove from poolpool.demote_bottom()
10Curriculum phase transition (warmup → exploration → exploitation)Curriculum.on_generation()
11Save checkpoint (pool + metadata + latest.json pointer)pool.save()

Distillation & Reward Tuning

▼

🚨 CHIMERA Diagnostics

Runs 7 checks every generation. Detects overfitting, excessive drawdown, insufficient trades, stagnation, negative Sharpe, low win rate.

  • → increase_drawdown_penalty
  • → decrease_drawdown_penalty
  • → tighten_risk / loosen_risk
  • → prioritize_consistency
  • → restore_defaults
▼

🔧 Auto Reward Tuner

Consumes mutations every 5 generations. Net vote per parameter. Bounded ±20% adjustments. Clamps to schema limits.

ParameterBounds
drawdown_penalty[0.0, 2.0]
transaction_penalty[0.0, 1.0]
holding_penalty[0.0, 1.0]
pnl_bonus_weight[0.5, 20.0]
reward_scale[10.0, 500.0]
▼

💰 Differential Sharpe Reward

Moody & Saffell (2001) incremental Sharpe via EMA. Applied every environment step.

total = sharpe_reward + pnl_bonus + dd_penalty + tc_penalty + hold_penalty

Effective weights = base_params × regime_multipliers

🌍 Regime Multipliers

Applied multiplicatively on every reward computation. Set by GeopoliticsExpert classification.

get_multipliers(regime)

▼

🤖 Agent Learning

Updated reward signals flow through SB3 .learn() calls. Agents adapt their policy networks to the modified reward landscape. This is how diagnostics ultimately shape agent behavior — through the reward function, not direct weight manipulation.

📚 Inverse RL (13F Distillation)

MaxEnt IRL infers reward weights from SEC 13F hedge fund filings. Extracts 12 features (Fama-French betas, sector deltas, HHI concentration, turnover, drawdown proxy) and maps them to RewardConfig parameters via regularized feature matching.

🛡 ATHENA Statistical Validation

Walk-Forward (WFE)
OOS Sharpe / IS Sharpe across 4 windows. Detects overfitting. Threshold: ≥ 0.40
PSR (Probabilistic Sharpe)
P(true SR > benchmark). Adjusts for skewness + kurtosis. Bailey & Lopez de Prado (2012).
DSR (Deflated Sharpe)
Adjusts for multiple testing bias (many agents tested). Bailey & Lopez de Prado (2014).
CHIMERA Fitness
Sharpe(35%) + MaxDD(20%) + PF(20%) + WFE(15%) + Consistency(10%) × stability
Pass thresholds: Sharpe ≥ 0.3 • Max DD ≤ 25% • Win rate ≥ 40% • Profit factor ≥ 1.1 • WFE ≥ 0.40

Market Regime System

▼

The GeopoliticsExpert classifies the macro regime from live news headlines (NewsAPI + Finnhub → Groq LLM analysis). Each regime applies multiplicative overlays to the base reward parameters, fundamentally changing how agents are trained.

🟢 RISK ON
Markets bullish, low VIX, positive economic data. Normal training parameters — Sharpe-driven with mild P&L bonus.
Drawdown penalty×0.8
Transaction penalty×1.0
Holding penalty×1.0
P&L bonus×1.2
Reward scale×1.0
🟡 RISK OFF
Uncertainty rising, moderate VIX, mixed data. Tighten drawdown limits, increase holding penalty, reduce P&L bonus.
Drawdown penalty×1.5
Transaction penalty×0.8
Holding penalty×1.2
P&L bonus×0.7
Reward scale×0.9
🔴 CRISIS
Black swan events, high VIX, market panic. Maximum conservatism — extreme drawdown protection, minimal position sizes, capital preservation mode.
Drawdown penalty×2.5
Transaction penalty×0.5
Holding penalty×1.5
P&L bonus×0.3
Reward scale×0.7
🟣 ANTIFRAGILE
Taleb barbell strategy. Chaos with clear directional signals (wars, oil shocks, tariffs). Protect the core aggressively, swing hard on asymmetric bets. Let winners run.
Drawdown penalty×2.0
Transaction penalty×1.3
Holding penalty×0.6
P&L bonus×1.8
Reward scale×1.1

HydraCorp (LLM-Powered Oversight)

▼

An AI-managed corporation layer that wraps the RL training pipeline. Corp agents analyze results, classify regimes, review proposals, and manage risk. Powered by Groq (Llama 3.3 70B, free tier) with rule-based fallbacks.

👑 Chief of Staff
Central Pipeline Orchestrator
Coordinates all corp agents. Pre-flight config validation. Routes pipeline results to strategy division. Manages shadow trading promotion. Produces CEO briefing.
Cost: $0 (orchestration only)
🌍 Geopolitics Expert
News-Driven Regime Classification
Fetches headlines from NewsAPI + Finnhub (138+ headlines). Groq LLM classifies regime (risk_on/risk_off/crisis/antifragile). Sector bias scoring. Ticker change recommendations. Runs daily.
Cost: ~1 Groq call/day (free tier)
💼 Hedge Fund Director
LLM Strategy Advisor
Analyzes pipeline results: fitness scores, Sharpe ratios, drawdowns. Generates strategy memos with config change proposals. Ticker intelligence evaluation. Auto-submits proposals with confidence scores.
Cost: ~1 Groq call/pipeline run
😈 Contrarian (Taleb)
Adversarial Reviewer
Fires when agents pass ATHENA. Mandated to find flaws. Fragility score 0.0-1.0 (antifragile/moderate/fragile). Proposes stress-test configs with tighter constraints. Verdict: fragile | antifragile | inconclusive.
Cost: ~1 Groq call/trigger
🔍 Innovation Scout
Tool & Technique Discovery
Weekly search for new libraries, techniques, research papers. Evaluates relevance, integration effort, priority. Categories: library, technique, research, data_source. Produces innovation briefs.
Cost: ~1 Groq call/week
🚨 Risk Manager
Circuit Breaker Enforcement
Processes CHIMERA diagnostic actions: alert, reduce_allocation (25-50% cut), flatten_positions, shutdown_agent. Backtesting: log only. Forward test: enforce. Caps reductions at 90% min allocation.
Cost: $0 (pure Python)
🔎 Operations Monitor
Training Watchdog
Runs every generation. Scans for anti-patterns (divergence, stagnation, overfitting, exploding rewards). Auto-fixes when safe. Submits proposals for manual review. Health score 0.0-1.0.
Cost: $0 (pure Python)
💻 Senior Dev
Config Regression Prevention
Maintains blacklist of failed configs (SHA256, Parquet-backed). Pre-pipeline validation. Post-pipeline regression check (>50% perf drop). Ticker patch review: obs space, churn, sector concentration. Deduplication.
Cost: $0 (pure Python)
👤 CEO Interface
Natural Language Commands
Chat interface for human operator. Commands: add/remove tickers, tighten/loosen risk, show status, approve/reject proposals. LLM intent parsing with regex fast-path. Sector switching presets.
Cost: ~1 Groq call/command
